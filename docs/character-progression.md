# Legacy.EXE Character Progression System

Legacy.EXE uses a **Compile Path** character system. Users do not just track habits. They build a visual character that evolves as they complete real-life quests.

The goal is simple:

> You are not leveling up a character. You are compiling yourself.

---

## Core Concept

Every user starts as **The Uncompiled**.

As users complete quests, earn XP, and level up, their character changes visually. Clothing, gear, weapons, tools, pets, auras, and backgrounds become rewards for real-life consistency.

This makes the app feel less like a checklist and more like a personal RPG.

---

## Compile Stages

## Stage 0: Uncompiled

**Levels:** 1–10

**Visual Style:**
- Simple dark hoodie
- Basic pants
- Minimal glow
- Starter shoes
- No major gear
- Blank-slate character energy

**Meaning:**
The user has started. They are no longer only thinking about change. They are taking action.

**Unlocks:**
- Starter avatar
- Starter title
- Basic daily quests
- First XP bar

**Possible Titles:**
- The Uncompiled
- Starter
- Wanderer
- Initiate

---

## Stage 1: Early Compile

**Levels:** 11–20

**Visual Style:**
- Glowing arm piece
- Better hoodie or jacket
- First visible accessory
- Small aura effect
- Light cybernetic detail

**Earned By:**
- Completing the first weekly challenge
- Building a 3–7 day streak
- Finishing beginner quests

**Reward Examples:**
- Cyber arm
- Focus gloves
- Starter weapon/tool
- Light aura
- Profile frame

**Possible Titles:**
- Initiate
- Early Compile
- Momentum Keeper
- First Build

---

**Levels:** 21–30

- More confident stance

**Earned By:**

- Hitting workout, study, project, or life goals

**Reward Examples:**
- Velocity boots
- Utility belt
- Cyber jacket
- Reinforced armor pieces
- Stat-themed aura

- Builder

- Completing harder challenges
- Maintaining long-term consistency

**Reward Examples:**
- Guardian cloak
- Neural spine effect
- Energy visor
- Advanced profile frame
- Rare weapon/tool cosmetics

**Possible Titles:**
- Ascendant
- The Consistent
- Boss Breaker
- Protocol Walker

---

## Stage 4: Full Compile

**Levels:** 41–50

**Visual Style:**
- Legendary outfit
- Full aura

# Compile Path Screen

The app should include a dedicated screen called **Compile Path**.

This screen shows the user’s current stage and future unlocks.

Example layout:

```text
Stage 0: Uncompiled        Level 1-10
Stage 1: Early Compile     Level 11-20
Stage 2: Mid Compile       Level 21-30
Stage 3: Advanced Compile  Level 31-40
Stage 4: Full Compile      Level 41-50
```

Each stage should show:
- Required level
- Required quests
- Next unlock
- Reward preview
- Current progress

Example:

```text
Next Unlock: Cyber Arm
Requirement: Reach Level 11
Progress: Level 7 / 11
Bonus: Complete 5 Daily Quests
```

# MVP Implementation Notes

For the first version, use simple 2D layered avatars instead of 3D characters.

Recommended layers:
- Base body
- Hair
- Face
- Top
- Bottom
- Shoes
- Hands
- Back item
- Weapon/tool
- Aura
- Pet

This makes it easier to swap gear as rewards without needing a full game engine.

---

# Design Direction

Visual style:
- Dark cyber-fantasy
- Hoodie-to-hero progression
- Blue compile glow as the default effect
- Gold for legacy and discipline rewards
- Purple for creativity
- Green for growth
- Red/orange for strength

The character should feel like they are evolving from an everyday person into a symbolic version of who the user is becoming.
